Purpose

The reason for using style sheets and having you follow these rules for composing HTML is to give all our documentation a consistent look and feel. Style sheets also make it possible to change aspects of the design for all the documents by modifying only a single document that contains the style rules. Therefore, the more layout is allowed to be controlled by the style sheet, rather than the internal markup of each page, the more the look and feel can be easily modified and kept consistent.

The way style sheets achieve this is by defining how the HTML markup tags will behave in a separate document that all the pages refer to. In this case the document is called doc_styles.css and it is referred to in the HEAD section of each document by this line:

<link rel=stylesheet type="text/css"
href="http://www.sci.utah.edu/scirun_docs/doc/doc_styles.css">

Style sheets should change the way you think about constructing HTML. Rather than using tags to develop the appearance of the individual elements within a document, you can use them to identify the basic parts of documents that can then be formatted by the style sheet. For example, rather that creating large headings here, small headings there, indent this bit here, we can create one tag called "head" for all main headings that describes font, size, indentation, color and all else. Then we define other tags for the Table of Contents, paragraphs, code examples, etc. If we stick to these predefined tags, all of our documents have a consistant look and any modifications to the style sheet will work for all the documents.

So, as you write you documents, try to fit each piece into a predefined catagory that one tag or class can do all the formatting for.

General Rules

  • Do not use a web page generator to edit the pages. Use a plain text editor such as EMACS or NOTEPAD. Web page editors that generate the HTML for you do not know how the styles are being used and will not generate correctly formatted documents for our purposes.

  • Place the statement <link rel=stylesheet type="text/css" href="http://www.sci.utah.edu/scirun_docs/doc/doc_styles.css"> in the HEAD section of the document.

  • Always use the beginning and end tags with the content in between. <tag> content... </tag>. That even goes for the <p> tag which should always have an associated </p> tag. This allows the styles to be applied only to the content in between the tags. The exceptions are the tags that have no end tag such as <img ...> and <br>.

  • Avoid using tags to modify the positioning of content such as placing <p>'s and <br>'s between paragraphs. The <p> tag is supposed to indicate the beginning of a paragraph and that is how it should be used. If used properly, the spacing issues can be handled by the style sheet. If you must, use a <p>&lt/p> to create one line of space between elements. For example, to separate items in a list.

  • If right or left brackets >< are to be displayed in your page, always replace the right brackets with &lt;, left brackets with &gt;, and & symbols with &amp;. The easiest way to achieve this is by using Search and Replace.

Using The Style Sheet

Conventions

Once the line that references the style sheet has been placed in your page, the styles will be affecting your tags. These styles are applied in two different ways; by the tag or by the class. Styles applied to a tag are specific to that particular tag. Styles applied to a class can be applied to any tag that includes the parameter class="classname".

By convention, the items in the Table of Contents will be numbered but the headers they refer to will not. This makes it less complicated to add and remove sections from the document. The class="toc" parameter used within the <ol> tag for the Table of Contents will automatically put in the roman numerals/numbers so they do not need to be included in the item text.

Another convention is that first paragraphs in each section will not be indented but following paragraphs will be. Two classes have been defined to take care of this, "firstpara" and "nextpara" respectively. Always use <p class="firstpara"> for the first paragraph in each section and <p class="nextpara"> for successive paragraphs.

The "example" class has been developed to format examples of computer text in monospace font. This is used to demonstrate bits of code or something typed on the computer. It can also be used in conjunction with the <pre> tag as <pre class="example"> to display ascii art or prestructured text.

To add space around images, place them between a set of <p> tags. For example:

<p><img ...></p>

Tags and Class Definitions

Tag or ClassDescription
<body>Since the <body> tag contains all of the content for the entire page, it can be used to set general parameters for the page. In this case it sets the background to a gentle grey and the default font to Arial, Helvetica, or Sans-serif, (depending on what their browser has) and establishes the margins on the left and right of 80 pixels.
<p>Generally used for bits of text which are not paragraphs and should not be indented or formatting other elements such as images. Gives the element a left margin of 100 pixels, right margin of 80 pixels and applies an Arial font. (Although this is the default font, it had to be applied to the individual tag styles to insure it's use.)
class="toc"Used for the Table Of Contents, generally applied to the <ol> tag, as <ol class="toc">. It applies roman numerals to the list (in IE), indents that section by 80 pixels from the paragraphs and applies an Arial font.
class="subtoc"Used for the Table Of Contents when a list within a list is needed to indicate subsections. Generally applied to the <ol> tag which is within the primary list for the Table of Contents. It maintains Arial/Helvetica font and applies disc shaped bullets./td>
class="title"Used to format the title text of the page. Generally applied to the <p> tag, as <p class="title>. Applies a large Times font to the text and centers it on the page.
class="subtitle"Used for smaller text which appears just below the title such as the "Last Update.." statement. Applies a bold Times font to the text and centers it on the page
class="head"Used for primary section titles. Generally applied to the <p> tag. Places the text on a blue bar extending across the screen, applies a large Arial/Helvetica font, indents the text by 80 pixels and places a 30 pixel margin below the bar.
class="subhead"Used for subsection titles. Generally applied to the <p> tag. Places the text on a blue bar extending across the screen, applies a bold Arial font, indents the text by 80 pixels and places a 30 pixel margin below the bar.
class="subsubhead"Used for third level subsection titles. Generally applied to the <p> tag. Places the text on a grey bar extending across the screen, applies a bold Arial font, indents the text by 80 pixels and places a 30 pixel margin below the bar.
class="firstpara"Used for first paragraphs in a section or subsection. Always applied to the <p> tag. Gives the text a left margin of 100 pixels and right margin of 80 pixels. No indentation is applied.
class="nextpara"Used for successive paragraphs after the first paragraph in a section or subsection. Always applied to the <p> tag. Gives the text a left margin of 100 pixels and right margin of 80 pixels. An indentation of 2em's is also applied.
class="example"Generally applied to the <p> tag or the <pre> tag. Used to display an example of code in Monospace font. If used with the <pre> tag, it maintains the spacing and line breaks of the original text so that blocks of code can be displayed as is. Applies a 50 pixel left margin indent from paragraphs and monospace font. No right margin specified.
<blockquote>Used to isolate a block of text as in a quotation. Gives the block a 50 pixel left margin indent from paragraphs and applies a Times font.
class="list"Used in conjuction with the <ol> or <ul> tags. Formats the list margins to be consistant with the document.
tdSimply insures that the text in a table cell will have Arial/Helvetica font.
class="small"Used to display text in a smaller font for notes, etc. Generally applied to the <p> tag. Applies same margins as paragraphs and small font size.
