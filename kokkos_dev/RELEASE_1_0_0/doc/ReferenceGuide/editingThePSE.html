Overview

SCIRun source code is stored in a CVS repository. CVS is a file revision control tool which facilitates multiple users developing the same source codes at the same time. It maintains a history of what has changed in a file, who made the change, and when it occurred, and allows for tagging a specific level of codes as a version release.

In order to check out a copy of the SCIRun source code from the repository, you must do the following:

  • Have a CS computer account created for you. Get access to csf.cs.utah.edu. Be added to the appropriate unix groups (csafe, scirun, biopse, etc.) See the complete New User's Checklist for more details.

  • Set the following unix environment variables. You will probably want to do this in you .cshrc file so that they will be set each time you log in:
    • setenv CVSROOT csf.cs.utah.edu:/csafe_noexport/cvs/cvsroot
    • setenv CVS_RSH ssh

  • Create a working directory for the copy of the files that you will be editing.

  • Change directories to the working directory.

  • NOTE: When running cvs commands, you may be querried for your password, depending on whether you have a .rhosts/.shosts file and where you are logging in from. If so, just type in your password. The examples below do not show a password query.

  • Checkout all or part of the repository that you are interested in. To do this, run the following CVS command: "cvs checkout SCIRun".

CVS

Once you have a copy of the source tree, you are free to edit and check in source files using CVS. Here are the main CVS commands you will likely use:
  • cvs update [-d]: When typed from within your working directory, this command will copy all files that have been updated in the repository (usually by other people) into your working directory. Any files that you have also modified will be marked with the differences. The "-d" argument is optional, and tells CVS to bring in new directories. By default, new directories added to the repository are not added to your working directory.

  • cvs status : Tells you about .

  • cvs log : Gives you the update log on .

  • cvs diff : Shows you the differences between in your working directory, and the most recent version in the repository.
If you are interested in being notified when any source files are modified, you may use majordomo to subscribe to either (or both) of "scirun-cvs" or "csafe-cvs" mailing lists.

Send mail to majordomo@cs.utah.edu with the following in the body of the e-mail:

subscribe scirun-cvs

Possible Problems and Their Solutions

  • You get "Permission Denied" when checking out or updating your working directory. This is (usually) caused when someone has added a new directory to the repository. When a new directory is created, CVS (incorrectly) sets the permissions to the defaults of directory's creator. Either a superuser or the owner of the directory may go to the cvs repository on csf and fix the file permissions.
