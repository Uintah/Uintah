Setting Up a New Developer

Purpose

This page is meant to be a complete list of steps required to get a new developer (fictitious account name newdev)

New Developer Todo:

  • Add account for newdev
  • Add account on csf.cs.utah.edu and have newdev added to the "csafe" group. To do this, send a request to some_appropriate_support_type_person. Once the account for newdev has been set up, newdev will be sent a temporary password.
  • Make sure newdev account has sufficient quota (how much?)
    • Grad Student quota is: xG
    • Staff Quota is: yG
    • Send mail tosome_appropriate_support_type_person and request the appropriate quota change for newdev
    • Suggest where newdev can create his/her build i.e. /home/sci/data1
  • Add to sci and scirun group
  • Add to sci and scirun-develop mailing list by mailing Majordomo.cs.utah.edu with a message like this:

subscribe sci newdev@cs.utah.edu subscribe scirun-develop newdev@cs.utah.edu

  • Add sci-food?
  • Mail newdev with link to help documentation to get them started. Also include the following info:

On gate.cs.utah.edu (the CS firewall) Use the command: passwd
On csf.cs.utah.edu (the C-SAFE server) Use the command: yppasswd
