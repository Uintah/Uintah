Chapter Overview

In this first chapter, we will construct a SCIRun network [link to 4.0 UG working with networks] to perform a visualization task: rendering of a tetrahedral mesh. Our "net," will consist of three modules: data reader, geometry converter, and renderer.

We will read the Field data from a file, set some rendering properties for the nodes, edges, and faces (the nodes will be rendered as blue spheres), and render the geometry to the screen in an interactive Viewer window.

To begin, we will construct a net from scratch (as shown, completed, in Figure 1.1). In the chapters that follow, we will expand on this network, adding more features and functionality as we progress.

We have just loaded the SCIRun Field dataset. A Field contains a geometric mesh and a collection of data values mapped onto that mesh. In this case, we use a tetrahedral mesh with voltages defined at the nodes of the mesh. The brief overview below describes the various types of geometric meshes, data values, and mappings that SCIRun supports.

Overview of field types

SCIRun has eight different geometric meshes available for Fields:

PointCloud: unconnected points.

ScanlineField: a regularly segmented straight line. (a regular 1D grid).

ContourField: a segmented curve.

ImageField: a regular 2D grid.

TriSurf: a surface made of connected triangles.

QuadSurf: a surface make of connected quadrillaterals.

LatticeVol: a regular 3D grid.

TetVol: a subdivision of space into tetrahedral elements.

HexVol: a subdivision of space into hexagonal elements.

In a field, there are, at most, four different places in which data can be stored: nodes, edges, faces, and cells. The dimensionality of the mesh type limits the available storage locations. For instance, a TriSurf has nodes, edges, and planar faces, but not cells, which we assume to be three-dimensional elements. As a result, a TriSurf cannot store any data in cells--but it can store data in edges or faces.

The following data types can be stored in a field:

  • Tensor
  • Vector
  • double precision
  • float(ing point)
  • int(eger)
  • short (integer)
  • char
  • unsigned int(eger)
  • unsigned short (integer)
  • unsigned char


ShowField module

We will now add a second module to our network. Select ShowField from the SCIRun/Visualization menu. (Use this module to visualize various field types).

Hooking modules together

You will now connect the two modules in your canvas so that data can flow between them. The Output port in SCIRun is always on the bottom of the module, whereas the Input ports are always on top. Each datatype in SCIRun that can pass through ports has a unique color. You may connect ports of the same color.

To hook the modules together:

  1. Position the mouse's pointer over the yellow output port on the FieldReader (Figure 1.7.1). Now press and hold down the mouse's middle button. You will see the name of the port and lines that indicate possible connections.
  2. Drag your middle mouse toward the ShowField input Field port (yellow).
  3. When the line turns red (Figure 1.7.2), release the mouse button.
  4. You should see a yellow bar showing a data flow connection between FieldReader and ShowField (Figure 1.7.3).
  5. If you ever make an incorrect connection, just click on that connection with the right mouse button and start again.

Setting the ShowField User Interface

ShowField has a User Interface (UI) option for rendering a field's geometer, and special options for rendering vector data. ShowField can use color to represent data within the geometry, if desired. For now, we will render blue spheres as the nodes of the geometry.

  1. Select the UI button on the ShowField module(Figure 1.8).


Figure 1.8

We will display the Field's nodes as spheres and set their color to blue:

  1. Select the Default Color button near the bottom of the dialogue to change the default color.
  2. When a new dialogue with red, green, and blue sliders appears, set red and green to 0.0, and blue to 1.00.
  3. Select the "OK" button in this color dialogue.
  4. Select "Cancel" to remove the dialogue from view.
  5. Change the Node Scale to 2 and set the Sphere Resolution to 5.
  6. Turn off the display of edges and faces.
  7. Select the Edges tab in the ShowField dialogue.
  8. Turn off Show Edges.
  9. Select the Faces tab.
  10. Turn off Show Faces.
  11. The ShowField module is now ready to render the nodes as blue spheres.
  12. Next, press the "Execute" button on the ShowField UI that forces the module to execute its algorithm.

ShowField works with any of the SCIRun Field data types. However, you will notice a delay when ShowField is asked to visualize a Field data type for the first time--this is because the data type must be first compiled into a shared library (which is saved for future runs of SCIRun). See Dynamic Compilation for more information.

Navigating the Viewer Window

Since the Viewer is one of the most important modules of SCIRun, you may want to know more about its functionality. The User's Guide [link to UG 5.0 Viewer] provides a full description of the Viewer.

Figure 1.10 shows the Viewer window from the network you have built earlier in this chapter. In the lower right corner of the Viewer's window you will find buttons for controlling the display. However, the controls you will use most frequently are not visible.

Mouse controls

The mouse has functionality that works with the Viewer. To get a feel for this functionality:

  1. Move the mouse to the center of the Viewer window.
  2. Click and hold the left mouse button.
  3. While holding, slide the mouse and objects in the Viewer window shift to follow the mouse movement.
  4. Now release the mouse and move it back to the center of the object.
  5. Click and hold the middle mouse button and then drag it around; objects in the Viewer will now rotate.
  6. Release the button and move the mouse to the edge of the objects in the Viewer window.
  7. Click, hold, and drag the middle mouse button. The object still rotates but in a different direction from before. Up and down motions cause the objects to rotate counterclockwise and clockwise, rather than around a horizontal axis by clicking on the middle of the objects.

To picture the rotation action of the mouse, imagine the objects sitting within a sphere. To grab a point on the sphere, click the middle mouse button. Moving the mouse drags that point on the sphere around as if the sphere were free to rotate but not to shift its position. By this mechanism, the actual movement of the objects depends on the point you click on before moving the mouse. To use an analogy, this is like placing a floating ball in a bowl full of water, touching the ball only at one point, and then rotating the ball by moving your finger.

Using this type of mouse control, objects can rotate in three dimensions while the mouse moves only in two. With experimentation, you will be able to orient objects in whatever direction you wish through simple mouse manipulations.

For more information, you may consult the User's Guide [link to UG 5.3].

Clicking and dragging the right mouse button causes the contents of the Viewer window to get larger and smaller. The creates the impression of moving closer in or farther away from the contents of the window (Figure 1.10.1).

Now we will use the controls in the Viewer window below the objects (Figure 1.10.2). Here, you see buttons for the following functions:

Saving and reloading networks[link to UG 4.13]

The three-module network that we created earlier provides the foundation for the nets we create through the rest of this tutorial. Now we will examine how to save nets to disk and reload them in a future SCIRun session.

To save a SCIRun network (Figure 1.11):

  1. Pull down the File menu from the top of the SCIRun window.
  2. Select "Save".
  3. When the file browser appears, follow the prompt to choose a location and filename for your net (by convention, we keep most nets in the SCIRun/src/nets/ directory, but you can store them in whatever location you wish).
  4. For this example, store the net as SCIRun/src/nets/show-torso-mesh.net (we use the .net suffix for SCIRun network files).


Figure 1.11


  1. Now exit SCIRun and restart the program.
  2. To load the saved net (Figure 1.12), pull down the "Load..." option from the File menu bar.
  3. Select SCIRun/src/nets/show-torso-mesh.net


Figure 1.12


The net should reload into SCIRun, where it was when you saved it. After changing anything in the display (e.g., rotating the view in the Viewer Window or changing the rendering color of the nodes in ShowField), then there are two options to re-save the net:

  • Overwrite the existing show-torso-mesh.net file by simply pulling down File > Save from the menu.
  • Alternatively, save it to a new file using File > Save As...

For more information on the File menu options (such as Insert and Clear), please refer to the User's Guide.
