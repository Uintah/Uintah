TBONP

Description:

Template class (Please read!)

Parallel implementation of the Temporal Branch-on-Need Octree (T-BON) for rectilinear scalar fields (see the thesis for more details).

Some preprocessing must be done before isosurface extraction. This preprocess step is DIFFERENT from that of the BONO module, but the SAME as that for the TBON module. See that documentation for details. During execution (run the module on an execution metafile), the user controls isovalue and time value sliders. Nodes and data are read in whenever needed to construct the current isosurface, otherwise left on disk. The data files are read in in blocks to improve I/O performance (again, see the thesis). A Marching Cubes-type interpolation is performed to create triangles for rendering.

A centralized work queue model is used to distribute computation. First, the tree and data for the current isosurface are read into memory. Then, the tree is traversed and all cells containing portions of the isosurface are enqueued. Threads then request jobs and are given batches of cells on which to perform interpolation and triangle construction.

The number of worker threads is hard coded. This is because I didn't want to deal with recreating the work queue and forking more threads whenever the user changed the "# processors" slider. So if you want to change it, just look in TbonP.cc for "np = " and change that number.

Metafile format:

Same as for the TBON module.
