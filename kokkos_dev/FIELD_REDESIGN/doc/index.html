General Information

SCIRun: A Scientific Programming Environment for Computational Steering (1995) describes the original PSE (SCIRun).

The SCIRun Computational Steering Software System discusses the design, implementation, and application of SCIRun.

The SCI group has published many other papers.

Existing Problem Solving Environments (PSEs):

  • SCIRun is a general purpose computational science problem solving environment developed by the Scientific Computing and Imaging Institute.
  • BioPSE is a problem solving environment for bioelectric field problems. It is developed by the NIH Center for Bioelectric Field Modeling, Simulation, and Visualization. BioPSE
  • Uintah is a problem solving environment for simulating fires involving a diverse range of accident scenarios including multiple high-energy devices, complex building/surroundings geometries and many fuel sources. It is developed by the Center for the Simulation of Accidental Fires and Explosions (C-SAFE).

All PSE's allow interaction between the modeling, computation, and visualization phases of a simulation. All PSEs are extensible.

Writing Code

Module writers must follow a specification procedure when writing new modules. Modules which are not fully spec'ed will not be incorporated into official SCIRun releases.

New modules are created with SCIRun's built in " Module Maker " software.

Coding Standards have been developed in order to provide a basis for consistent, easily reviewable, and maintainable code. Your source code must be commented (documented) according to the SCIRun standard.

SCIRun source code, documentation, and other files, are managed by the Concurrent Versions System software. Developers must be familiar with its use in the SCIRun project. Also see the cvs information prepared by the Uintah project.

You may browse SCIRun's ABI (compliments of cocoon).

Data converters are standalone programs that convert "foreign" data to an existing SCIRun datatype object (SCIRun datatypes are "persistent" - data objects can be saved to a file and later recreated from the content of the file). You may need to write one some day.

Tck/Tk Info

SCIRun graphical user interfaces are composed with the scripting language Tcl and its extensions Tk, iTcl, iTk, and BLT.

There are a number of good books describing tcl/tk. One is Practical Programming in Tcl and Tk by Brent B. Welch.

A new book explaining itcl (aka [incr Tcl/Tk]) is [incr Tcl/Tk] from the GROUND UP by Chad Smith.

Man pages for GUI widgets from the standard tcl and itcl libraries are available online.
