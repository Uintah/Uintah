Validation Tool for SCIRun XML Files

Introduction

This tool may be used to validate various types of XML files, e.g.

  • XML Module descriptions
  • XHTML files
  • DocBook documents

Basically this tool determines if a document is:

  1. A well formed XML document (e.g. no missing end tags).
  2. Follows the rules of the document's language (which is specified in the document's <!DOCTYPE ...> declaration).

If your document is not valid then this tool will report a series of error messages otherwise it just reports some document statistics (e.g. number of elements).

Note: Multi-file documents are not yet handled correctly.

Please read Notes section below.

Notes

  • You may recieve fatal and/or non-fatal errors. Fatal errors terminate the validation while non-fatal errors do not.

  • Non-fatal errors are usually related to an incomplete document (missing elements/content -- please add missing content, especially for module descriptions).

  • Fatal errors are usually due to violations of XML composition rules.

  • Note that fatal error messages are often misleading. You may need to search around a bit to find the real location of the problem.

  • Multi-file documents are not yet handled correctly.

  • This utility expects the document's DTD to be specified in the <DOCTYPE...> declaration via an http url in either a PUBLIC or SYSTEM identifier, as in:

    <!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Transitional//EN"
"http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd">

    or

    <!DOCTYPE html SYSTEM
"http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd">

    XML Module descriptions are handled specially though. They don't use an http type url. They are expected to contain a <DOCTYPE...> declaration though, something like this:

    <!DOCTYPE html SYSTEM "../../../doc/Utilities/XML/component.dtd">
