Description of MD simulation parameters

Please note that lines beginning with the "*" character are comments


Cutoff Radius
The cutoff radius represents the distance (in Angstroms) at which the dispersion interactions are truncated to zero. The appropriate pressure correction term due to the cutoff of dispersion interactions (Cr-6) is included. Corrections to the energy due to the truncation are not calculated. No shifting or cutting of the potential is implemented. Max"Drift":
The max drift represents the cumulative maximum displacement (in Angstroms) since the previous neighbor list calculation which is allowed to occur for any molecular before a new neighbor list calculation is required.

Delta(fs)
Delta is the smallest integration time step.

Number of Steps
The total number of integration time steps for the MD run. If a multiple time step integrator is used, the the number of steps represents the total overall number of the largest time steps which occur during the simulation.

Temparature
Temperature is the desired temperature (in Kelvin) for NPT and NVT runs that use the Nose-Hoover thermostat, as well as the set temperature for IsoKinetic simulations.

Pressure
Pressure is the desired simulation pressure in atmospheres for NPT runs that use the Nose-Hoover barostat.

Num_NHC
Num_NHC is the number of individual coupled thermostats in the thermostatting system. The coupling method is that outlined by Martyna, Klein, and Tuckerman in their Nose-Hoover Chains technique (J. Chem. Phys. 97(4), 2635 (1992))

OmegaQ and OmegaW
OmegaQ and OmegaW represent the frequencies (in 1/fs) of the Nose-Hoover thermostat and barostat.

Initialize(fs)
The Initialize variable specifies the amount of time (in femtoseconds) for initialization of the system before variables are counted toward contribution to the overall statistics of the system (output in fort.70) begins.

Average(fs)
The Average variable specifies the length of time block (in femtoseconds) over which thermodynamic quantities are averaged for reporting in fort.65.

Pressure(fs)
The Pressure variable specifies the time interval (in femtoseconds) between calculations of the system stress tensor and overall pressure. This time should be an integral multiple of the largest time step.

Energy Split(fs)
The Energy Split value determines the frequency of calculation of detailed statistics which include information about the individual contributions of energetic sources within the system to the overall energy of the system. This information has no impact on the evolution of the simulation, and is generally only necessary for debugging or tracking the evolution of the energy in specific cases. For that reason, a large value is recommended for this paramater in most situations, in order to avoid spending excess time on the calculation of these statistics.

Coord Outputs(fs)
This setting determines the time interval between outputting the instantaneous coordinates and velocities to the file coords.out. This file serves as a system checkpoint, saving the current overall configuration of the system when output.

Binary Output Flags
A set of four binary values which determine whether or not to output the following binary format output files:
  • BoxDipole(76) - Average dipole moment of the molecules in the system.
  • Coordinates(77) - Coordinates vector of each force center in the system.
  • Stress Tensor(78) - Stress tensor of the overall system.
  • Velocities(79) - Velocity vector of each force center in the system.
Each file is output as fort.XX where XX is equal to the number in parentheses after the output variable name.

Trajectory Output Frequency
Controls the respective output frequency for each of the above file types. Output frequencies are listed in femtoseconds, and should be integral multiples of the largest time step in the system.

Resize Box?
In NVT and NVE runs the box can be rescaled to change the overall box volume (and system density) from the initial size set from the input files. The Target Box Size field determines the final system box side length. The rate of change is therefore the difference between the initial and final sizes, divided by the number of simulation steps.

Output Additional Information?
If true, this option toggles screen output of additional information such as instantaneous pressure, temperature, stress tensor, etc. at every simulation step.

Calc Intermolecular?
This parameter allows for toggling on/off intermolecular interations. Setting this parameter to false allows for a gas-phase type calculation, where individual molecules do not interact with one another. For liquid simulations, or simulations of realistic gases where intermolecular interactions are important, this parameter should be set to true.

Exclude 1-4?
This parameter controls inclusion of 1-(2-3)-4 nonbonded interactions in the force field. Most of the explicit atom force fields include 1-4 nonbonded interaction terms, so this parameter should be set to false.

Isokinetic Ensemble?
This parameter toggles the use of the IsoKinetic ensemble to control temperature. If strict rescaling of the temperature to the target value via simple velocity scaling is desired, this variable should be set to true. Enabling this option necessarily overrides Nose-Hoover thermostat control.

Reduce 1-4(Exc. mu-mu)?
If this parameter is set to true, all nonbonded 1-4 interactions (except for dipole-dipole) are reduced by the Reduce 1-4 Factor. The reduction implementation is consistent with those commonly incorporated in AMBER and OPLS-AA force fields.

Reduce Q-mu 1-4?
Setting this parameter to true enables the reduction of static charge-induced dipole nonbonded 1-4 interactions by the Red. Q-mu Factor without affecting other nonbonded 1-4 interactions.

Constrain Bonds?
If true, this enables the SHAKE algorithm for constraining bonds, with the maximum average constrained bond length deviation set by the value in Tolerance (SHAKE) with units of 10-20 m2.

Electrostatic Calculation Type
Lucretius uses an Ewald summation technique for the calculation of long range electrostatics. The parameter Alpha represents the screening length parameter for the Ewald summation, while the Maximum K Vector dictates the maximum magnitude of the longest inverse lattice vector employe in the Ewald calculation.

Integrator:
Lucretius implements the RESPA integrator with a three level multiple time step breakdown. There are three adjustable components of the 3 time step integrator:
  • Medium Steps - Number of small steps (Delta - see line 2) performed for every medium step performed.
  • Large Steps - Number of small steps (Delta) performed for every large step performed.
A single small time step (Delta) includes calculation and integration of bonds, bends, and improper torsions.
A single medium time step includes the above, plus torsions and short-range nonbonded interactions (with cutoff radius set by Medium Timestep Cutoff).
A single large time step includes the above (with nonbonded interactions being performed with the full, rather than reduced, cutoff radius), plus long range electrostatics, polarizability calculations, and any extended lagrangian (e.g. thermostatted) contributions present.

Polarization Tol.
Tolerance threshold under which the induced many-body polarization values of the polarizable atoms are considered to be self-consistent. This value is in units of A2e2

Taper Start:
Induced dipole-induced dipole interactions are scaled from nominal values to zero to prevent discontinuous forces arising from polarizable interactions. Rescaling of the induced dipole-induced dipole interaction begins at the radial value set by Taper Start and is driven to zero at the distance specified by the Cutoff Radius.
