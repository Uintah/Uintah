Chapter Overview

Chapter one demonstrates the construction of a SCIRun network that performs a visualization task; rendering a tetrahedral mesh. The net (network) consists of three types of modules: data reader, geometry converter, and renderer. These modules are FieldReader, ShowField, and Viewer.

This chapter instructs the user on reading the Field data from a file, setting the rendering properties for the nodes, edges, and faces (the nodes are rendered as blue spheres), and rendering the geometry to the screen in an interactive ViewWindow.

A module is a single-purpose unit that functions within a dataflow environment. Modules always have at least one input port for receiving data, located at the top of the module, or one output port for sending data, located at the bottom of the module.

For a more complete description of modules, consult the User's Guide, Section 2.3 "Modules and Networks."

Each dataflow datatype in SCIRun has a unique color such as yellow, purple, or pink. Ports of the same color can be connected with a datatype. These colors are described in detail in the User's Guide, Section 4.2, "Anatomy of a Module".

By joining the modules together, data passes through the pipe from one module to the next; two or more connected modules form a SCIRun network, or net.

All modules have an indicator that tells the user if messages exist in a module's log. Different colors represent different types of messages. Blue represents a Remark (as the SampleField module indicates the number of seed points), yellow for Warning. If the module's indicator is clicked, the log will open with a message.

A Field contains a geometric mesh and a collection of data values mapped on to that mesh. Data can be stored at the nodes, edges, faces, and cells of the mesh. In this case, use a tetrahedral mesh with voltages defined at the nodes of the mesh.

The dimensionality of the mesh type limits the available storage locations. For example, TriSurf has nodes, edges, and planar faces, but not cells, which are assumed to be three-dimensional elements. As a result, a TriSurf cannot store any data in cells--but can store data in edges or faces.

The overview below describes various types of geometric meshes, data values, and mappings that SCIRun supports.


Overview of Field types

SCIRun has eight geometric meshes available for Fields:

PointCloudMesh: unconnected points

ScanlineMesh:a regularly segmented straight line (a regular 1D grid)

CurveMesh: a segmented curve

ImageMesh: a regular 2D grid (see note below)

TriSurfMesh: a surface made of connected triangles

QuadSurfMesh: a surface made of connected quadrilaterals

LatVolMesh: a regular 3D grid

TetVolMesh: a subdivision of space into tetrahedral elements

HexVolMesh: a subdivision of space into hexagonal elements

NOTE: ImageMesh is not used for image processing since SCIRun does not support that function.



The following data types can be stored in a Field:

  • tensor
  • vector
  • double precision
  • floating point
  • integer
  • short integer
  • char
  • unsigned integer
  • unsigned short integer
  • unsigned char

Dynamic Compilation

ShowField works with all SCIRun Field data types. These data types must first be compiled into a shared library (as needed) where they are saved for future use. This process is called "Dynamic Compilation."

As a result, Dynamic Compilation causes a delay when the module executes for the first time. For more details, refer to the PDF document "Dynamic Compilation."

In addition, the module turns light yellow while it compiles, and all output goes to the module's log as a remark. Please note that sgi machines may take more time before the green status bar appears.

Setting the ShowField User Interface

The ShowField module has options for changing the visual representations of a Field's geometry. For example, in the dataset just loaded (using the FieldReader module), change the color of the nodes to blue spheres.

  1. Select the UI button on the ShowField module.

  2. Select the Default Color button near the bottom of the GUI to change the default color and a separate color GUI appears.

  3. In the color GUI, use the sliders to adjust the color values. Set Red and Green to 0.00, and Blue to 1.00. The Alpha value remains at 0.50.

  4. Select the OK button in the color GUI. Notice that the Default Color swatch in the ShowField GUI has changed to blue. The SCIRun environment should look like Figure 1.10.

  5. In the color GUI, select Cancel to close the window.


Figure 1.10: SCIRun environment after changing ShowField's Default Color.

Now change the scale and resolution of the nodes.

  1. In the ShowField GUI, a slider widget represents the Node Scale. The Node Scale interval can be increased or decreased by a power of 10 by pressing the + or - button. Set the Node Scale to 2.

  2. Set the Cylinder and Sphere Resolution to 5.

  3. Go to the Edges tab and turn off the display of edges by deselecting the "Show Edges" check box.

  4. Go the the Faces tab and repeat the same action.

  5. Close the ShowField GUI by pressing the Dismiss button.

The ShowField module is ready to render the nodes as blue spheres. The module Interactively Updates, by default, to execute after every change. Or, select the Execute Button Only box to contain all changes until the Execute button is pressed. Consult the User's Guide 4.5, "Executing a Network", for more details.

Mouse controls

In the Viewer, the mouse can be used to rotate, scale, and translate the image. . The User's Guide 5.2, "Mouse Control in the Viewer" provides more complete information.

Translating the image

  1. Move the mouse to the center of the image.

  2. Click and hold the left mouse button.

  3. Move the mouse in order to move the image.

  4. Release the button and the image stays where it was placed in the ViewWindow.

Rotating the image

  1. Click and hold the middle mouse button.

  2. Move the mouse in order to rotate the image.

  3. Release the mouse button.

  4. If the mouse button is released while dragging, the image will continue to rotate.

  5. Click the middle mouse button to stop rotation.

Scaling the scene

  1. Click the right mouse button and drag the mouse.

  2. Move the mouse up or to the left to zoom the image out.

  3. Move the mouse down or to the right to zoom the image in.

Saving and reloading networks

Having created a three-module network, the net can be saved to disk. This net may be reloaded in a future SCIRun session.

Saving a SCIRun network:

  1. Click on the File menu and select "Save As."

  2. When the file browser appears, follow the prompt to choose a location and filename for the net. By convention, most nets are stored in the SCIRun/src/nets/ directory, but can be stored in another location.

  3. For this example, store the net as SCIRun/src/nets/show-torso-mesh.net, as in Figure 1.15. The .net suffix represents a SCIRun network file.

Please note that, to avoid losing work, it is strongly recommended that nets are frequently saved.



Figure 1.15: "Save As" GUI.

  1. Click the Save button. The network will be saved, and the dialog disappears.

  2. Exit SCIRun.

Loading a SCIRun network:

  1. Start SCIRun.

  2. From the File drop-down menu, select the the Load... option.

  3. Select SCIRun/src/nets/show-torso-mesh.net.


Figure 1.16: GUI for loading a network

The net reloads into SCIRun, where it was previously saved. If the net was saved with any of the module UIs open, those UIs automatically re-open when loaded to the net. After changing any module settings (e.g., rotating the image in the ViewWindow or changing the rendering color of the nodes in ShowField), there are two options for re-saving the net:

  1. Overwrite the existing show-torso-mesh.net file by using File/Save from the drop-down menu.

  2. Or, save the net to a new file by using File/Save As...
