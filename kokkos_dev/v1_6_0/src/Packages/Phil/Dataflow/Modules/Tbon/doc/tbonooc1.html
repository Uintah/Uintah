TBONOOC1

Description:

Template class (Please read!)

Implementation of an out-of-core algorithm based on the Temporal Branch-on-Need Octree (T-BON) for rectilinear scalar fields.

Some preprocessing must be done before isosurface extraction. This preprocessing is DIFFERENT from that of the TBON and TBONOOC2 modules - don't just use the files resulting from those preprocess. Your results will be incorrect.

During execution (run the module on an execution metafile), the user controls isovalue and time value sliders. Nodes and data are read in whenever needed to construct the current isosurface, otherwise left on disk. Nodes and data are read in in blocks to improve I/O performance (again, see the thesis). Dynamic data structures are used to hold both node and data blocks (see algorithm 1 in the thesis). A Marching Cubes-type interpolation is performed to create triangles for rendering.

The maximum sizes for the dynamic data structures are declared statically in TbonOOC1.cc:

static const int NODEBRICKS = < size of node list >
static const int DATABRICKS = < size of data list >

The minimum settings for these values are:

static const int NODEBRICKS = 1;
static const int DATABRICKS = 8;
These are the minimum settings! I don't recommend setting DATABRICKS to anything higher, anyway.

Metafile format:

Although the results of preprocessing are not the same, the metafile format for TBONOOC1 is the same as that of the TBON module.
