TBONOOC2

Description:

Template class (Please read!)

Implementation of another out-of-core algorithm based on the Temporal Branch-on-Need Octree (T-BON) for rectilinear scalar fields.

Some preprocessing must be done before isosurface extraction. This preprocessing is DIFFERENT from that of the TBON and TBONOOC1 modules - don't just use the files resulting from those preprocess. Your results will be incorrect.

During execution (run the module on an execution metafile), the user controls isovalue and time value sliders. Nodes and data are read in whenever needed to construct the current isosurface, otherwise left on disk. Nodes and data are read in in blocks to improve I/O performance (again, see the thesis). A static data structure holds the nodes, while a dynamic data structure is used to hold data blocks (see algorithm 2 in the thesis). A Marching Cubes-type interpolation is performed to create triangles for rendering.

The maximum size for the dynamic data structure that holds the data bricks is declared statically in TbonOOC2.cc:

static const int DATABRICKS = < size of data list >

The minimum size is 8.

Metafile format:

Although the results of preprocessing are not the same, the metafile format for TBONOOC2 is the same as that of the TBON module.
