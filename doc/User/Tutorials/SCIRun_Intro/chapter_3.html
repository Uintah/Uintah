Chapter Overview

Chapters 1 and 2 demonstrate the construction of SCIRun networks to visualize the geometry and data values in a Field. This chapter explores additional visualization techniques, and examines filter modules that receive a Field as input, and output a newly derived Field.

Rather than building one net to introduce these modules, several nets will be constructed. Each net will describe a single visualization technique. This chapter is divided into three sections, one for each method.

3A: FieldBoundary

First, perform a boundary extraction. The FieldBoundary module extracts a boundary surface from a volume. For example, given a TetVolMesh, the module extracts the boundary faces and outputs a TriSurfMesh.

Create a net using the FieldBoundary module (SCIRun/Fields/FieldBoundary):

  1. Add a FieldReader module to the canvas, and load the utahtorso-lowres-voltage.tvd.fld file.

  2. Also add a FieldBoundary module from SCIRun/Fields/FieldBoundary.

  3. Attach the output from a FieldReader into the FieldBoundary input port.

  4. Add a ShowField module and connect the FieldBoundary's output to the ShowField's input.

  5. Set the ShowField module to display edges only (turn off the nodes and the faces).

  6. In the SHowField UI, render the edges as a cylinder, with a scale of 1.0.

  7. Connect the output from the ShowField module into a Viewer.

  8. Press the Autoview button and rotate the image downward, to look like Figure 3.2.

The net should look like Figure 3.1. Save this net as 3A.net, it will be used in Chapter 4 of this tutorial, Putting the nets together.

3B:DirectInterpolate

Next, interpolate data values from one Field to another Field. This section, demonstrates how to load two datasets:

  • The first contains the utahtorso-lowres-voltage.tvd.fld that we examined in Chapter 1, which we will refer to as the volume Field.

  • The second contains the electrode positions in utahtorso-lowres-electrodes.pcd.fld, from Chapter 2.

Now that the datasets are loaded, create a new Field by interpolating the electrode positions and voltage values.

Interpolation is the means by which we calculate the value at a certain point by using the known values at surrounding points. For each electrode, we must find the corresponding voltage value from the volume Field.

The DirectInterpolate module implements this algorithm (SCIRun/Fields/DirectInterpolate). The following modules are needed to run the DirectInterpolate net:

  • Two FieldReader modules
  • One DirectInterpolate module
  • One ShowField module
  • One GenStandardColorMaps module
  • One RescaleColorMap module
  • One Viewer module

These modules should be hooked together as shown in Figure 3.3. NOTE: be sure to set the Node Scale in the ShowField UI to be at least 4-6, and hit the Autoview button in the ViewWindow.


Figure 3.3, showing the DirectInterpolate net

Also, make sure to rotate the image in the ViewWindow downward as it appears in Figure 3.4. The FieldReader on the left should read in the utahtorso-lowres-voltage.tvd.fld input file, and the FieldReader on the right should read in the utahtorso-lowres-electrodes.pcd.fld input file.

3C: StreamLines

Streamline advection is the last visualization technique explored in this chapter. The StreamLines module visualizes vector Fields by integrating curves through a gradient vector flow Field.

Generate the input vector Field by using the Gradient module (SCIRun/Fields/Gradient) in order to compute the gradient of the utahtorso-lowres-voltage.tvd.fld.

The SampleField module will generate a sample distribution of StreamLine seed points. SampleField generates samples from any type of Field input and outputs the samples as a PointCloud Field. The following modules are needed to run the StreamLines net:

  • One FieldReader module
  • One Gradient module
  • One SampleField module
  • Two ShowField modules
  • One DirectInterpolate module
  • One GenStandardColorMaps module
  • One RescaleColorMap module
  • One Viewer module
  • One StreamLines module

These modules should be hooked together as shown in Figure 3.5.


Figure 3.5, showing the StreamLines net

The SampleField module can generate samples randomly or the user can select samples using a 3D widget. For Streamline visualization, use a 3D rake widget for distributing seed points. The user can interactively position the rake of seed points within the volume:

  • From the SampleField UI, select the rake option and set the number of samples to 30.

To attach scalar data values to the StreamLine nodes, set the following options:

  • With the FieldReader, select the input file utahtorso-lowres-voltage.tvd.fld.

  • In the ShowField UI, display only Nodes, either as spheres or points, with NodeScale set to approximately 1--2.

  • For the second ShowField module lower-left in Figure 3.5, set the UI to display Edges only as cylinders with a Cylinder Scale of 2.

The StreamLines module works by integrating the seed points through the vector Field. A fifth order integration method is used; the Runge-Kutta-Fehlberg algorithm. Three parameters must be to run this algorithm: Error Tolerance, Step Size, and a Maximum Number of Steps. Set these three parameters, respectively, to 0.01, 0.1, and 200. The other UI options can remain at their default settings.

Controlling Widgets

The rake is one of many widgets used in SCIRun. Widgets are graphical elements in the Viewer that allow the user to interactively control features of the display. The SCIRun widgets have similar visual characteristics, but manipulate different aspects of the display, as detailed in the User's Guide 6.5, Control Widgets.

Control widgets by:

  • Press and hold the shift key, left-click the mouse on the widget and move it around the region of view in order to change its location and action.

  • Press and hold the shift key, left-click the mouse on a sphere in the widget and change the position of that widget.

  • Press and hold the shift key, left-click the mouse on the end knob (green resize cylinders) in the widget and change, for example, the density of streamlines.

  • SCIRun redraws the image in the ViewWindow when the mouse button is released.

Network execution and data dependence

There are several ways to execute a SCIRun network that has not fired.

In the StreamLines network:

  • On the module itself, right-click the mouse and select the Execute option from the dropdown menu (Figure 3.7).

  • Or, in several modules, such as ShowField and StreamLines, the UI has its own Execute option.

These actions will execute the selected module, and any modules that depend on that module.


Figure 3.7


After executing the StreamLines network, an image like Figure 3.8 should appear in the ViewWindow. The rake widget will appear as a gray bar with spheres at the ends. The user's image may be somewhat different than Figure 3.8, but the user should see the rake and colored streamlines. If the streamlines do not appear, the user may need to check the object list and turn on the Edges.
