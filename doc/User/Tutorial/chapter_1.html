Chapter Overview

In this first chapter, we will construct a SCIRun network that performs a visualization task: rendering a tetrahedral mesh. Our net (network) will consist of three types of modules: data reader, geometry converter, and renderer. These modules are FieldReader, ShowField, and Viewer.

We will read the Field data from a file, set the rendering properties for the nodes, edges, and faces (the nodes will be rendered as blue spheres), and render the geometry to the screen in an interactive ViewWindow.

To begin, we will construct a net as shown completed in Figure 1.0 and, in the chapters that follow, we will expand on this network, adding more features and functionality as we progress.

FieldReader User Interface

You should set two environment variables in your shell before starting SCIRun (for more information about environment variables, you may consult the User's Guide 3.4, "SCIRUN_DATA"). This process assumes that you have downloaded the SCIRunData directory, a separate download from the SCIRun build.

  1. The SCIRUN_DATA variable points to the location of the SCIRunData directory on your system.

  2. The SCIRUN_DATASET variable indicates which dataset to load since many of our example networks run with a variety of data inputs, e.g., sphere, brain-EG, and utahtorso-lowres.

To begin, we will construct a network that loads a geometric mesh from disk and renders it to the screen. In your Unix shell window:

  1. Set your SCIRUN_DATA variable, pointing to the location of SCIRunData (as shown in Figure 1.1). The "path_to_data" should represent the directory path where you have stored the data on your computer.

  2. Set your SCIRUN_DATASET variable to utahtorso-lowres.

The NetworkEditor screen consists of three frames: the GlobalView frame in the left uppermost corner; the Error frame in the right uppermost corner, and the NetEdit frame. The NetEdit frame is the large area in which you will construct your network of modules.

Since the NetEdit frame is so extensive, the GlobalView frame shows you which part of the NetEdit frame you are currently viewing. The Error frame will report any errors during startup.

A module is a single-purpose unit that functions within a dataflow environment. Modules will always have at least one input port for receiving data, located at the top of the module, or one output port for sending data, located at the bottom.

For a more complete description of modules, you may consult the User's Guide, Section 2.3 "Modules and Networks."

Each dataflow datatype in SCIRun has a unique color such as yellow, purple, or pink. You may connect ports of the same color with a datatype. These colors are described in more detail in the User's Guide, Section 4.2, "Anatomy of a Module".

By joining the modules together, data passes through the pipe from one module to the next; two or more connected modules form a SCIRun network, or net.

All modules have an indicator that tells the user whether any messages exist in a module's log. The different colors represent different types of messages. Blue represents a Remark (as the SampleField module indicates the number of seed points), yellow for Warning. If you click the module's indicator, the log will open with a message.

A Field contains a geometric mesh and a collection of data values mapped on to that mesh. Data can be stored at the nodes, edges, faces, and cells of the mesh. In this case, we use a tetrahedral mesh with voltages defined at the nodes of the mesh.

The dimensionality of the mesh type limits the available storage locations. For example, TriSurf has nodes, edges, and planar faces, but not cells, which we assume to be three-dimensional elements. As a result, a TriSurf cannot store any data in cells--but it can store data in edges or faces.

The brief overview below describes the various types of geometric meshes, data values, and mappings that SCIRun supports.


Overview of Field types

SCIRun has eight different geometric meshes available for Fields:

PointCloudMesh: unconnected points.

ScanlineMesh:a regularly segmented straight line (a regular 1D grid).

CurveMesh: a segmented curve.

ImageMesh: a regular 2D grid.(see note below)

TriSurfMesh: a surface made of connected triangles.

QuadSurfMesh: a surface made of connected quadrilaterals.

LatVolMesh: a regular 3D grid.

TetVolMesh: a subdivision of space into tetrahedral elements.

HexVolMesh: a subdivision of space into hexagonal elements.

NOTE: ImageMesh is not used for image processing since SCIRun does not support that function.



The following data types can be stored in a Field:

  • tensor
  • vector
  • double precision
  • floating point
  • integer
  • short integer
  • char
  • unsigned integer
  • unsigned short integer
  • unsigned char

Dynamic Compilation

ShowField works with any of the SCIRun Field data types. These data types must be first compiled into a shared library, as needed, where they are saved for future use. This process is called "Dynamic Compilation."

As a result, Dynamic Compilation causes a delay when the module executes for the first time. For more detail about this process, refer to the PDF document "Dynamic Compilation."

In addition, the module will turn light yellow while it compiles and all output will go to the module's log as a remark. Please note that sgi machines may take more time before the green status bar appears.

Setting the ShowField User Interface

The ShowField module has options for changing the visual representations of a Field's geometry. For example, in the dataset we have just loaded (using the FieldReader module), you will change the color of the nodes to blue spheres.

  1. Select the UI button on the ShowField module.

  2. Select the Default Color button near the bottom of the GUI to change the default color and a separate color GUI appears.

  3. In the color GUI, you can use the sliders to adjust the color values. Set Red and Green to 0.00, and Blue to 1.00. The Alpha value remains at 0.50.

  4. Select the OK button in the color GUI. Notice that the Default Color swatch in the ShowField GUI has changed to blue. Your SCIRun environment should look like Figure 1.10.

  5. In the color GUI, select Cancel to close the window.


Figure 1.10: SCIRun environment after changing ShowField's Default Color.

We will now change the scale and resolution of the nodes.

  1. In the ShowField GUI, a slider widget represents the Node Scale. You can increase or decrease the Node Scale interval by a power of 10 by pressing the + or - button. Set the Node Scale to 2.

  2. Set the Cylinder and Sphere Resolution to 5.

  3. Go to the Edges tab and turn off the display of edges by deselecting the "Show Edges" check box.

  4. Go the the Faces tab and repeat the same action.

  5. Close the ShowField GUI by pressing the Dismiss button.

The ShowField module is now ready to render the nodes as blue spheres. The module will Interactively Update, by default, to execute after every change. Or, you may select the Execute Button Only box to contain all changes until you press the Execute button. Consult the User's Guide 4.5, "Executing a Network", for more details.

Viewer module

The Viewer is the last module that we will add to our network. You may find more information about the Viewer in the User's Guide 5.0, "Visualization with the Viewer."

  1. Locate the Viewer under the SCIRun/Render menu.

  2. Connect the output port from ShowField into the Viewer input port. The Viewer module now has a new input port; an example of a SCIRun module that has dynamic input ports.

  3. Open the ViewWindow by pressing the Viewer's UI button.

  4. In the ViewWindow, you will see a set of axes, representing X, Y, and Z directions. Press the Autoview button near the bottom of the window, and you should see the points of the geometry, similar to Figure 1.11 .

Mouse controls

In the Viewer, you can rotate, scale, and translate the image using your mouse. The User's Guide 5.2, "Mouse Control in the Viewer" provides more complete information.

Translating the image

  1. Move the mouse to the center of the image.

  2. Click and hold the left mouse button.

  3. Move the mouse in order to move the image.

  4. Release the button and the image will stay where you have placed it within the ViewWindow.

Rotating the image

  1. Click and hold the middle mouse button.

  2. Move the mouse in order to rotate the image.

  3. Release the mouse button.

  4. If you release the mouse button while dragging, the image will continue to rotate.

  5. Click the middle mouse button to stop rotation.

Scaling the scene

  1. Click the right mouse button and drag the mouse.

  2. Move the mouse up or to the left in order to zoom the image out.

  3. Move the mouse down or to the right in order to zoom the image in.

Saving and reloading networks

Having created a three-module network, you can now save the net to disk. This net may be reloaded in a future SCIRun session.

Saving a SCIRun network:

  1. Click on the File menu and select "Save As."

  2. When the file browser appears, follow the prompt to choose a location and filename for your net. By convention, most nets are stored in the SCIRun/src/nets/ directory, but you may store them in another location.

  3. For this example, store the net as SCIRun/src/nets/show-torso-mesh.net, as in Figure 1.15. The .net suffix represents a SCIRun network file.

Please note that we strongly recommend that you frequently save your nets to avoid losing work.



Figure 1.15: "Save As" GUI.

  1. Click the Save button. The network will be saved, and the dialog disappears.

  2. Exit SCIRun.

Loading a SCIRun network:

  1. Start SCIRun.

  2. From the File drop-down menu, select the the Load... option.

  3. Select SCIRun/src/nets/show-torso-mesh.net.


Figure 1.16: GUI for loading a network

The net will reload into SCIRun, where you previously saved it. If you saved the net with any of the module UIs open, those UIs automatically re-open when you load the net. After changing any module settings (e.g., rotating the image in the ViewWindow or changing the rendering color of the nodes in ShowField), you have two options for re-saving the net:

  1. Overwrite the existing show-torso-mesh.net file by using File/Save from the drop-down menu.

  2. Or, you can save the net to a new file by using File/Save As...

Summary

In this chapter, we created a three module network to visualize the geometry of a mesh. We rendered that mesh to the viewing window, and practiced saving and reloading networks.
