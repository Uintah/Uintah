Chapter Overview

In Chapters 1 and 2, we constructed SCIRun networks to visualize the geometry and data values in a Field. In this Chapter, we will explore more visualization techniques, and look at "filter" modules that input a Field, and produce a new derived Field as output.

You can also find more information about visualization technniques in the User's Guide [link UG ?].

Rather than building one net to introduce all of these new modules, we will construct several nets, each describing a single technique. To keep things simple, we have subdivided this chapter into three sections, one for each method.

3A: FieldBoundary

We will first implement will be a boundary extraction. This module takes any type of Field as input, and produces a new Field that only contains the boundary as output. For example, given a tetrahedral volume Field, the module produces a triangulated surface Field of just the boundary faces as output.

To create a FieldBounday module from SCIRun/Fields/FieldBoundary:

  1. Attach the output from a FieldReader (choose the utahtorso-lowres-voltage.tvd.fld file again) to the FieldBoundary input port.
  2. Pass the contents of the first output port of the FieldBoundary module into the first input port of a ShowField module.
  3. Set the ShowField module to display just edges (turn off the nodes and the faces).
  4. Render the edges as a cylinder, with a scale of 1.0.
  5. Connect the output from the ShowField module into a Viewer.

You net should look like Figure 3.1; the view in the ViewWindow should look like Figure 3.2.

3B:DirectInterpolate

Another useful technique for visualizing data is to interpolate data values from one Field to another Field. In this section, we will read in two datasets:

  • The first contains the utahtorso volume mesh we looked at in Chapter 1.
  • The second contains the electrodes that we looked at in Chapter 2 [link here to chapter 2, electrodes].

The volume Field contains simulated voltage values through the entire volume. For each position of the electrodes, we must look up the corresponding voltage value from the volume mesh.

We can then output a new combined Field containing the geometry of the electrodes and their corresponding interpolated voltages from the volume.

The module that implements this algorithm is called DirectInterpolate (SCIRun/Fields/DirectInterpolate).

To implement the net described above, we will need:

  • two FieldReader modules
  • one DirectInterpolate module
  • one ShowField module
  • one GenStandardColorMaps module
  • one RescaleColorMap module
  • one Viewer module

These should be hooked together as shown in Figure 3.3.

The FieldReader on the left should read in the utahtorso-lowres-voltage.tvd.fld input file, and the FieldReader on the right should read in the utahtorso-lowres-electrodes.pcd.fld input file. The resulting image in the ViewWindow should look like Figure 3.4.

3C: StreamLines

Streamline advection is the last visualization technique we will explore in this Chapter. The StreamLine module takes in a vector Field, and a Field of sample points. The module then outputs a new Field that consists of a sequence of path-integral curves generated from advecting (integrating) the sample positions through the (derivative) vector Field.

We will generate the input vector Field by using the Gradient module (SCIRun/Fields/Gradient) to compute the gradient of the utahtorso-lowres-voltage.tvd.fld (this produces the negative electric field):

  1. Connect the output from the Gradient module to the the first Field input port of the StreamLines module.

To generate a sample distribution of streamline seed points, we will use the SampleField module. SampleField generates samples from any type of input field and outputs the samples as a PointCloud field.

The samples can be generated randomly or can be user selected via a 3D widget. For streamline visualization, we use a 3D "rake" widget to place our distribution of seed points.

To interactively position the rake of seed points within the volume:

  1. Connect the Geometry output port (pink) from this module to the Viewer module.
  2. Select the "rake" option from the SampleField UI and set the number of samples to 30.
  3. Connect the Field output port from this module to the second Field input port of the StreamLines module.

At this point, we have streamlines coming out of the StreamLines module, and we would like to render them. Before we do, we can attach scalar datavalues to the streamline nodes. To accomplish this, we use the DirectInterpolate module described above.

We will again use the voltage datavalues from the utahtorso-lowres-voltage.tvd.fld Field. Attaching all of the modules, we have the network shown in Figure 3.5.


Figure 3.5


The Streamlines module works by integrating the seed points through the vector Field. We use a fifth order integration method: Runge-Kutta-Fehlberg algorithm. To run this algorithm, we must set three parameters: Error Tolerance, Step Size, and a Maximum Number of Steps. These three parameters should be set to 0.01, 0.1, and 200, respectively, in the StreamLines UI (as shown in Figure 3.6).

Network execution and data dependence

There are several ways to execute a SCIRun network that has not yet "fired".

  1. Click the right mouse button on the name of one of the module icons.
  2. Select the "Execute" option from the dropdown menu (Figure 3.7).
  3. Press any of the "Execute" buttons that appear on many of the module User Interfaces.
    4. These actions will execute the selected module, as well as any modules that are part of its data dependency graph.


Figure 3.7


After executing the network, you should see an image like Figure 3.8 in the ViewWindow. The rake widget appears as a gray stick in this scene.


Figure 3.8


You can move this widget around to new positions by:

  1. Press the shift key and click on the widget with the left mouse button.
  2. Release the mouse button.

The streamlines will automatically recompute from the new seed positions.
